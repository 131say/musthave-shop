# Workflow: Ты + ChatGPT + Cursor (MustHave)

## Роли

### Ты
- Принимаешь решения.
- Пишешь, что хочешь изменить/добавить.
- Тестируешь сайт глазами пользователя.

### ChatGPT (я)
- Проектирую архитектуру.
- Пишу текстовые ТЗ.
- Даю diff-патчи, команды для терминала и подсказки для Cursor.
- Слежу за целостностью проекта.

### Cursor
- Применяет изменения в код.
- Создаёт и правит файлы.
- Показывает ошибки сборки/типа.

## Правила работы над кодом

1. **Один шаг за раз.**  
   Мы не пытаемся сделать всё сразу. Сначала БД, потом модели, потом API, потом UI.

2. **Все изменения по плану.**  
   Перед изменением кода — короткий план: что создаём/правим.

3. **Чистые коммиты.**  
   Логика по максимуму отделена:
   - domain/ (бизнес-логика),
   - app/ (страницы),
   - components/ (UI).

4. **Фокус на простоте.**  
   Код должен быть понятен тебе, даже если ты не программист: названия файлов и переменных говорящие, без «магии».

5. **Реферальная логика отдельно.**  
   Всё, что касается бонусов и рефералок:
   - отдельные функции/сервисы,
   - отдельные таблицы,
   - легко отключаемая/меняемая логика.

## Как использовать эти docs
- При любом новом шаге:
  - сверяемся с `PROJECT_OVERVIEW` и `PRODUCT_REQUIREMENTS`,
  - проверяем согласованность с `ARCHITECTURE` и `DOMAIN_MODEL`,
  - при изменениях рефералок — обновляем `REFERRAL_PROGRAM`.
